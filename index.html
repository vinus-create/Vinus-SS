            • 2️⃣

            Why Browser Mode?
            ✅ Works reliably — uses your real Shopee session
            ✅ No CAPTCHA — Shopee treats it as normal browsing
            ✅ Malaysian IP — no geo-blocking
            ❌ Server mode fails — Vercel servers are US-based, Shopee blocks them regardless of cookies

            Full Setup Guide

            Supabase + Vercel + Claude in Chrome — complete production stack

            1
            Supabase
            Create project, run SQL schema, copy URL + service key
            2
            GitHub
            Push shopee-scope folder, connect repo to Vercel
            3
            Vercel
            Add env vars, deploy — get your live dashboard URL
            4
            Scrape
            Open shopee.com.my in Chrome, run scraper.js via Claude
            5
            Analyse
            Open dashboard on phone, select shop, get insights
            1. Supabase Setup
            • 1️⃣
              Go to supabase.com → New Project
            • 2️⃣
              SQL Editor → paste supabase_schema.sql
            • 🔑
              Settings → API → copy Project URL and service_role key
              NOT the anon key — service key bypasses RLS
            • ✅
              Tables: shops, products, scrape_log
            2. Vercel Deploy
            • 1️⃣
              Push project to GitHub repo
            • 2️⃣
              vercel.com → Import Git Repository
            • 🔑
              Environment Variables:
              SUPABASE_URL=https://xxx.supabase.co
              SUPABASE_SERVICE_KEY=your_service_key
            • 🌐
              Your live URL: your-app.vercel.app
            3. Run Scraper
            • 1️⃣
              Open Chrome with Claude in Chrome extension
            • 2️⃣
              Navigate to shopee.com.my
            • 3️⃣
              Edit scraper.js — set VERCEL_URL + shop usernames
            • 4️⃣
              Ask Claude to run the scraper via javascript_tool
            • 📱
              Open dashboard on phone → select shop → done!
            🍪 Auto-Scrape Cookie Setup checking...

            The daily cron uses your Shopee session cookie to scrape real sales data. Save your cookie once — it lasts ~30–60 days, then refresh with one click.

            STEP 1 — Add the bookmarklet to your browser

            Drag this button to your bookmarks bar:

            🍪 Save Shopee Cookies

            ⚠️ You must drag it — clicking here won't work (security restriction)

            STEP 2 — Use the bookmarklet on Shopee
            1. Go to shopee.com.my and log in as vinusahboy
            2. Click the 🍪 Save Shopee Cookies bookmarklet in your bookmark bar
            3. A popup will say "Cookies saved!" — done!
            OR — Paste cookie string manually
            ℹ️ Cookies are stored encrypted in Supabase. The cron reads them server-side — your credentials never leave Supabase.
            🔁 Refresh cookies when the cron starts failing again (usually every 30–60 days).
